# Code Guide

This directory contains the Python source code used for the experimental part
of the **Game-Theoretic Statistics and Sequential Anytime-Valid Inference
(SAVI)** project.

## Directory Structure

```text
code/
├── confidence-sequence/
│   └── main.py                  # Quantile confidence sequence experiment
├── kelly-game/
│   └── main.py                  # Kelly betting and KL growth experiment
└── README.md                    # This file
```

The shared Python dependencies are listed in `../requirements.txt` at the
project root.

## Environment

Use Python 3.11 or newer. The pinned dependencies require a standard
scientific Python environment with NumPy, SciPy, and Matplotlib.

From the project root, create and activate a virtual environment:

```bash
python -m venv .venv
source .venv/bin/activate
```

On Windows PowerShell:

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
```

Install dependencies:

```bash
python -m pip install -r requirements.txt
```

Then move to the source directory:

```bash
cd code
```

If the virtual environment is created while already inside `code/`, use
`../requirements.txt` when installing dependencies.

## Files

| File | Description |
|---|---|
| `kelly-game/main.py` | Simulates one million tosses of a biased coin with default probability `p = 0.6`. It searches over betting fractions, compares the simulated optimum with the Kelly fraction, and verifies the connection between optimal log-wealth growth and Bernoulli KL divergence. |
| `confidence-sequence/main.py` | Builds a time-uniform confidence sequence for a fixed quantile and simultaneous confidence bands for a distribution's CDF. It also performs Monte Carlo coverage checks for fixed and multiple quantiles. |

## Running the Demos

Run all commands below from the `code/` directory.

### 1. Kelly Betting Game

```bash
python kelly-game/main.py
```

Expected terminal output includes:

- the theoretical Kelly fraction;
- the best fraction found by simulation;
- simulated and theoretical log-growth values;
- their differences from the Bernoulli KL divergence.

A Matplotlib window shows average log-wealth growth over the tested betting
fractions. With the default `p = 0.6`, the optimal fraction should be close to
`0.2`.

Default experiment settings:

| Setting | Value |
|---|---|
| Coin probability | `p = 0.6` |
| Number of tosses | `1,000,000` |
| Random seed | `42` |
| Betting-fraction grid | `0` to `0.999`, with `1,000` values |

### 2. Quantile Confidence Sequence

```bash
python confidence-sequence/main.py
```

Expected terminal output includes:

- coverage checks for the true `0.9` quantile at selected sample sizes;
- estimated time-uniform coverage for the fixed quantile;
- estimated simultaneous coverage over finite quantile and time grids.

A Matplotlib window displays:

1. The lower and upper confidence-sequence bounds for the `0.9` quantile.
2. Empirical CDF confidence bands at sample sizes `100`, `1,000`, and `10,000`.

Default experiment settings:

| Setting | Value |
|---|---|
| Distribution | Standard Cauchy |
| Confidence level | `95%` (`alpha = 0.05`) |
| Quantile level | `0.9` |
| Visualization sample size | `100,000` |
| Coverage sample size | `10,000` per trial |
| Monte Carlo trials | `300` |
| Random seed | `42` for the main visualization |

The script also supports standard normal, exponential, Student-t with three
degrees of freedom, and uniform distributions. Change `dist_name` in
`confidence-sequence/main.py` to one of:

```text
cauchy
normal
exponential
t
uniform
```

## Reproducibility Notes

- Every experiment uses fixed NumPy random seeds.
- No external datasets are required; all observations are generated by the
  scripts.
- Both scripts call `plt.show()` and therefore require a graphical Matplotlib
  backend.
- The confidence-sequence experiment runs 300 Monte Carlo trials by default
  and may take several minutes, depending on the machine.
- To make a quick test run, reduce `n_trials`, `sample_size`, or
  `coverage_sample_size` in `confidence-sequence/main.py`.
- Numerical output may vary slightly across Python, NumPy, and SciPy versions,
  but the qualitative conclusions should remain unchanged.

## Troubleshooting

If Python cannot import NumPy, SciPy, or Matplotlib, confirm that the virtual
environment is active and reinstall the root dependencies:

```bash
python -m pip install -r ../requirements.txt
```

If no plot window appears, run the scripts in a desktop environment with an
interactive Matplotlib backend. The numerical results are still printed to the
terminal before the plotting window opens.
